# Standardized BDD Acceptance Criteria Template

For every task, ensure the following format is used to validate the feature. The expectation is to avoid lazy coding and ensure robustness.

## Feature: [Feature Name]

**Scenario:** [Specific Scenario Description]
- **Given** [Initial context, precondition, or state]
- **When** [Action taken by the user or system]
- **Then** [Expected outcome, result, or state]
- **And** [Any additional expectations]

*Note: All tasks must be validated against their respective Gherkin scenarios prior to being marked as completed. Run automated tests or manual verification matching these scenarios.*
